## Ephemeral Commerce PWAs — Snapchat-style Storefronts

### The Concept

Personalized, single-use storefronts generated by an agent, delivered like a Snapchat message, and self-destructing after their purpose is fulfilled. The storefront is a PWA — no app install required. It disappears after purchase, after N views, or after a TTL expires.

### Why It Matters

Ephemerality increases conversion because the storefront signals: *this was made for you, it won’t be here tomorrow.* Generic storefronts have no advantage over Shopify or Gumroad. The moat is personalization — an agent generating a storefront customized for a specific recipient in seconds.

### How It Might Work

- Token stored as `tokens/{id}.json` in the repo with `expires`, `redeemed`, `maxViews` fields
- Deployed PWA checks token on load via the git provider API
- Payment via Stripe; webhook commits `redeemed: true` on purchase
- Delivery via link, QR code, SMS, NFC tap, or Bluetooth beacon
- Agent receives MMCP notification on open and on purchase

### Use Cases

- Digital downloads, video access tokens, event tickets
- Physical products (T-shirts, prints, merch)
- In-person sales via NFC tap at markets or events
- Content unlocks (pay to access a video, then the storefront is gone)

### Open Questions

- View counter enforcement: edge function or service worker?
- iOS NFC limitation: Web NFC is Android/Chrome only; QR fallback on iOS
- Stripe webhook endpoint: Cloudflare Worker or Vercel Edge?
